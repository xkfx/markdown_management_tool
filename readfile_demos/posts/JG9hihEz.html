1、git init——初始化仓库

初始化成功的标志如下（.git默认为隐藏）

2、git status——查看仓库状态

ps:未跟踪的文件 (untracked files)

不妨在尝试在仓库建立一个readme.md，然后再次git status,结果

3、git add——向暂存区中添加文件

untracked file都是不属于git仓库的版本管理对象。要想让文件成为git的管理对象，就需要执行git add，

将文件加入【暂存区】（提交前的一个临时区域）

（图为执行 git add readme.md后）

4、git commit——保存仓库的历史记录

git commmit命令可以讲当前暂存区中的文件实际保存到仓库的历史记录中。通过这些记录，我们就可以在工作树中复原文件。

-m 参数后的字符串为提交信息，是对提交的概述。

5、git log——查看提交日志

ps: commit 右边的是hash值

基础参数

git log readme.md 显示单一文件的提交信息

git log -p readme.md 显示文件改动
